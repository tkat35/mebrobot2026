01

Arduino Nedir ve Kod Nasıl Çalışır?

Arduino, üzerinde küçük bir bilgisayar çipi bulunan bir devre kartıdır. Robotu hareket ettirmek, sensörlerden veri okumak, LED yakmak gibi işlemleri bu kart sayesinde yaparsın. Kartı çalıştırmak için ona ne yapması gerektiğini bir program (kod) yazarak söylemen gerekir.

🧠 Neden Böyle?

Arduino C++ diline dayalı bir dil kullanır. Ama endişelenme — yarışma için gereken kısmı çok anlaşılır ve öğrenilebilir.

Her Arduino programının iki zorunlu parçası vardır:

Arduino C++ 
void setup() {
  // Başlangıç ayarları — sadece bir kez çalışır
  // Pinleri hazırla, kütüphaneleri başlat
}

void loop() {
  // Sürekli tekrar eder — robot kapanana kadar
  // Sensör oku, motoru sürdür, karar ver...
}
💡 Bunu Aklında Tut

setup() bir kez çalışır — kart açılınca. loop() ise durmadan döner. Robotun davranışının büyük çoğunluğunu loop() içine yazarsın.

Her satırın sonunda noktalı virgül (;) kullanman gerekir. Parantez { } arasındaki her şey o bloğa aittir. Yorum satırları // ile başlar ve kod çalıştırılmaz, sadece sana not bırakmak içindir.

✅ Ne Öğrendik?
  • setup() bir kez çalışır, loop() sürekli tekrar eder
  • Her satır ; ile biter
  • // yorum satırları çalıştırılmaz, not almak içindir

Değişkenler ve Veri Tipleri

Değişken, bir bilgiyi saklayan kutucuktur. Tıpkı oyunda skoru aklında tutman gibi — Arduino da bazı bilgileri değişkenlerde saklar. Her değişkenin bir tipi ve bir adı olmalıdır.

Söz Dizimi 
tip degiskenAdi = deger;

Veri Tipleri

Tip Ne Saklar? Örnek Değer Bellek
int Tam sayı -32768 ... 32767 2 byte
uint8_t 0–255 arası sayı 0, 100, 255 1 byte
float Ondalıklı sayı 3.14, -0.5 4 byte
bool Doğru/Yanlış true, false 1 byte
String Metin "Merhaba!" Değişken
💡 uint8_t Nedir?

uint8_t aslında "0 ile 255 arasında tam sayı" demektir. Arduino pinlerinin değerleri (0–255 arası PWM, HIGH/LOW) hep bu aralıktadır. Bu yüzden pin numaraları ve PWM değerleri için uint8_t kullanmak hem doğrudur hem de int'e göre daha az bellek kullanır. u = işaretsiz (eksi sayı yok), 8 = 8 bit, t = tip.

Arduino C++ 
int skor = 0;              // tam sayı
float mesafe = 23.5;       // ondalıklı
bool engelVar = false;     // evet/hayır
String mesaj = "Hazır!";   // metin
uint8_t hiz = 200;         // 0-255 arası hız değeri
uint8_t ledPin = 13;       // pin numarası için ideal

#define ile Sabit Tanımlama

Bazen bir değerin program boyunca hiç değişmeyeceğini bilirsin — mesela bir pine bağlı LED'in pin numarası. Bu durumda değişken yerine #define kullanabilirsin.

🧠 Neden #define?

Normal değişkenler Arduino'nun RAM belleğinde yer kaplar. #define ise bellek kullanmaz — kodu derlenmeden önce, tanımladığın ismi direkt sayıyla değiştiriri. Robotlarda bellek kıymetlidir. Ayrıca kodu okuyan biri 13 yerine LED_PIN görünce ne olduğunu hemen anlar.

❌ Sihirli sayı — anlaşılmaz

digitalWrite(13, HIGH);
analogWrite(9, 200);

✅ #define ile — net ve anlaşılır

#define LED_PIN 13
#define SOL_MOTOR 9
digitalWrite(LED_PIN, HIGH);
analogWrite(SOL_MOTOR, 200);
⚠️ Dikkat

#define ile tanımlanan sabitin değeri program çalışırken değiştirilemez. Eğer sonradan değişmesi gerekiyorsa const int veya normal değişken kullan.

✅ Ne Öğrendik?
  • Değişken = bilgi saklayan kutucuk, mutlaka tipi olmalı
  • uint8_t = 0–255 arası sayı, pin ve hız değerleri için idealdir
  • #define = sabit değer için, bellek kullanmaz, kodu açık eder

04

Dijital ve Analog Sinyal / Pin Kavramı

Arduino kartındaki her delik bir pin'dir. Pinlerden veri gönderip alırsın. Ama önce iki farklı sinyal türünü anlamak gerekir:

🔲 Dijital Sinyal

Sadece iki değer alır: HIGH (5V, açık) veya LOW (0V, kapalı). Ya var ya yok, arası yoktur. Buton, çizgi sensörü gibi bileşenler dijital sinyal verir.

〰️ Analog Sinyal

Sürekli değişen, sonsuz ara değer alabilir. Arduino'da 0–1023 arasında okunur. Sıcaklık sensörü (LM35), ışık sensörü (LDR) gibi bileşenler analog sinyal verir.

Arduino Pin Türleri

Pin TürüEtiketNe İçin?
Dijital Çıkış/GirişD2–D13LED yakma, buton okuma, motor sürücüsüne sinyal
PWM (analog benzeri çıkış)~3, ~5, ~6, ~9, ~10, ~11Motor hızı, LED parlaklığı (0–255)
Analog GirişA0–A5Sensörden değer okuma (0–1023)
🧠 PWM Nedir?

Arduino dijital pinler aslında tam analog değer üretemez. Bunun yerine çok hızlı açıp kapatarak "sanki analog gibi" bir sinyal üretir — buna PWM (Pulse Width Modulation) denir. Kartın üzerinde ~ işareti olan pinler PWM destekler. Motor hızını bu pinlerden kontrol edersin.

✅ Ne Öğrendik?
  • Dijital sinyal = 0 ya da 1, analog sinyal = sürekli değişen
  • Analog okuma = 0–1023, PWM yazma = 0–255
  • ~ işaretli pinler PWM destekler — motor hızı için bunları kullan
05

Koşul İfadeleri: if / else if / else

Robot karar vermelidir: "Önünde engel var mı? Çizgide mi?" Bu kararları if ile alırsın. "Eğer şu doğruysa bunu yap, değilse şunu yap" mantığıdır.

Söz Dizimi 
if (koşul) {
  // koşul doğruysa burası çalışır
} else if (başka_koşul) {
  // ilk koşul yanlış, bu doğruysa çalışır
} else {
  // hiçbiri doğru değilse burası çalışır
}

Gerçek bir robotik örneğiyle görelim:

Arduino C++ 
float mesafe = 15.0;  // sensörden okunan mesafe (cm)

if (mesafe < 10) {
  // 10 cm'den yakın engel var, dur!
  motorlariDurdur();
} else if (mesafe < 20) {
  // 10-20 cm arası, yavaşla
  yavasGit();
} else {
  // 20 cm'den uzak, normal git
  ileriGit();
}
💡 Koşulları Birleştirme

Birden fazla koşulu aynı anda kontrol edebilirsin:

Arduino C++ 
// HEM çizgi üstünde HEM mesafe güvenliyse git
if (cizgideмi == true && mesafe > 15) {
  ileriGit();
}

// Sol VEYA sağ sensör çizgi görüyorsa dön
if (solSensor == HIGH || sagSensor == HIGH) {
  don();
}
✅ Ne Öğrendik?
  • if ile koşullu karar alınır
  • else if ile birden fazla durum kontrol edilir
  • else hiçbiri tutmazsa çalışır
  • && (ve), || (veya) ile koşullar birleştirilebilir
06

Döngüler: for ve while

Aynı işi birden fazla kez yapmak istediğinde döngü kullanırsın. "5 kez yap", "engel geçene kadar dön" gibi durumlar için birebir.

for Döngüsü

for, kaç kez çalışacağı belli olan döngülerde kullanılır.

Söz Dizimi 
for (int i = 0; i < 5; i++) {
  // Bu blok 5 kez çalışır (i: 0, 1, 2, 3, 4)
}
🧠 Üç Parçayı Anla

int i = 0 → sayacı başlat  |  i < 5 → koşul doğruysa devam et  |  i++ → her turda sayacı bir artır (i++ = i + 1 kısaltması)

Arduino C++ — Gerçek Örnek 
// Robotun LED'ini 5 kez yakıp söndür
for (int i = 0; i < 5; i++) {
  digitalWrite(LED_PIN, HIGH);   // LED yak
  delay(300);                      // 300ms bekle
  digitalWrite(LED_PIN, LOW);    // LED söndür
  delay(300);
}

// 4 çizgi sensöründen sırayla değer oku ve yazdır
for (int i = 0; i < 4; i++) {
  int deger = digitalRead(cizgiPinleri[i]);
  Serial.println(deger);
}

while Döngüsü

while, ne zaman duracağı belli olmayan ama bir koşula bağlı döngülerde kullanılır.

Arduino C++ 
// Mesafe 20 cm'nin altında kaldığı sürece bekle
while (mesafe < 20) {
  mesafe = mesafeOku();  // yeniden ölç
  delay(50);
}
⚠️ Sonsuz Döngüye Dikkat

Koşul hiç false olmazsa döngü sonsuza kadar devam eder ve robot takılır. while içinde koşulu değiştirecek bir işlem mutlaka olmalıdır.

✅ Ne Öğrendik?
  • for → kaç kez tekrarlayacağı bilinen işlemler için
  • while → bir koşul sağlandığı sürece devam eden işlemler için
  • Her döngüde koşul güncellenmeli, yoksa robot takılır
07

Fonksiyon Tanımlama

Fonksiyon, bir işi yapan ve isim verdiğin kod bloğudur. Aynı işi defalarca yazman gerekmez — bir kez tanımlarsın, istediğin yerde çağırırsın.

🧠 Neden Fonksiyon?

Düşün: robot her engelde durmalı, birkaç saniye beklemeli, sonra geri gitmeli. Bunu her seferinde 10 satır yazmak yerine engeldenKac() diye bir fonksiyon yaz ve tek satırda çağır. Hem daha kısa hem de hatayı tek yerden düzeltirsin.

Söz Dizimi 
geri_donus_tipi fonksiyonAdi(parametre_tipi parametre) {
  // işlemler
  return deger; // geri dönüş varsa
}

Değer Döndürmeyen Fonksiyon (void)

Arduino C++ 
// Motorları tamamen durdur
void motorlariDurdur() {
  analogWrite(SOL_MOTOR, 0);
  analogWrite(SAG_MOTOR, 0);
}

// İstenen hızda ileri git (0-255)
void ileriGit(uint8_t hiz) {
  analogWrite(SOL_MOTOR, hiz);
  analogWrite(SAG_MOTOR, hiz);
}

// loop() içinde kullanım:
void loop() {
  ileriGit(200);        // 200 hızında git
  delay(2000);
  motorlariDurdur();    // dur
}

Değer Döndüren Fonksiyon

Arduino C++ 
// İki değerin ortalamasını hesapla ve döndür
float ortalama(float a, float b) {
  return (a + b) / 2.0;
}

// Kullanım:
float sonuc = ortalama(14.5, 16.3);
Serial.println(sonuc);  // 15.40 yazar
💡 Fonksiyon İsimlendirme

Fonksiyon isimleri ne yaptığını anlatmalı: ileriGit(), sagaDon(), mesafeOku() gibi. İlk harf küçük, her yeni kelime büyük harfle başlar — buna camelCase denir.

✅ Ne Öğrendik?
  • Fonksiyon = isimlendirilmiş, tekrar kullanılabilen kod bloğu
  • void = değer döndürmez; int/float/... = değer döndürür
  • Parametrelerle farklı değerler gönderilebilir
  • İyi isimlendirilmiş fonksiyonlar kodu okunaklı yapar
08

Temel Arduino Yapıları ve Bileşenler

pinMode — Pini Hazırla

Bir pini kullanmadan önce ona "giriş mi çıkış mı?" diye sorman gerekir. Bunu setup() içinde yaparsın.

Arduino C++ 
void setup() {
  pinMode(13, OUTPUT);   // 13 numaralı pin çıkış (LED yakacağız)
  pinMode(7,  INPUT);    // 7 numaralı pin giriş (buton okuyacağız)
}

digitalWrite & digitalRead

Arduino C++ 
digitalWrite(13, HIGH);   // pine 5V ver → LED yanar
digitalWrite(13, LOW);    // pine 0V ver → LED söner

int durum = digitalRead(7); // butonun durumunu oku (HIGH veya LOW)

analogWrite & analogRead

Arduino C++ 
analogWrite(9, 128);      // PWM: ~yarı hız (0=dur, 255=tam hız)
analogWrite(9, 0);         // dur
analogWrite(9, 255);       // tam hız

int deger = analogRead(A0); // A0 pininden oku (0–1023)

Serial — Bilgisayarla Konuş

Robotun ne yaptığını bilgisayar ekranında görmek için kullanılır. Hata ayıklarken (debug) çok işe yarar.

Arduino C++ 
void setup() {
  Serial.begin(9600);   // haberleşme hızını ayarla
}

void loop() {
  int deger = analogRead(A0);
  Serial.print("Sensör: ");
  Serial.println(deger);  // sonra yeni satır ekler
  delay(500);
}

delay ve millis — Zamanlama

delay(ms) — Basit Bekleme

delay(1000); // 1 saniye dur

Kullanımı kolay ama bu sürede robot hiçbir şey yapamaz — sensör okuyamaz, karar veremez.

millis() — Engelsiz Zamanlama

unsigned long baslangic = millis();
if (millis() - baslangic > 1000) {
  // 1 saniye geçti
}

Robot arka planda çalışmaya devam eder. Yarışmada daha profesyonel bir yaklaşım.

💡 Hangisini Kullan?

Başlangıçta delay() ile pratik yap. Robotun aynı anda birden fazla şey yapması gerektiğinde (sensör + motor + kontrol) millis()'e geç.

09

Kütüphane, Sınıf ve Nesne Kavramı

Bazı bileşenler çok karmaşık çalışır — onları sıfırdan programlamak yerine kütüphane kullanırsın. Kütüphane, başkasının senin için hazırladığı bir araç kutusudur.

🧠 Benzetme

Telefon kullanmak için içini bilmen gerekmez — açarsın, kullanırsın. Kütüphane de böyledir: kütüphaneyi eklersin, hazır komutlarını kullanırsın.

Kütüphaneler genellikle bir sınıf (class) içerir. Sınıf, o bileşenle ilgili tüm özellik ve komutları bir arada tutar. Sınıftan bir nesne (object) oluşturursun — bu artık senin o bileşenin kendi kopyasıdır.

Arduino C++ — Servo Örneği (kavram için) 
#include <Servo.h>         // 1. Kütüphaneyi ekle

Servo kolServo;             // 2. Servo sınıfından "kolServo" adında nesne oluştur
Servo kameraServo;          // 3. İkinci bir Servo nesnesi — aynı sınıftan 2 farklı nesne!

void setup() {
  kolServo.attach(9);       // kolServo → 9. pine bağlı
  kameraServo.attach(10);   // kameraServo → 10. pine bağlı
}

void loop() {
  kolServo.write(90);       // kol servosunu 90°'ye getir
  kameraServo.write(45);   // kamera servosunu 45°'ye getir
}
💡 Sınıf ve Nesne İlişkisi

Servo sınıf (şablon), kolServo ise o şablondan üretilmiş nesne (gerçek kopya). Tıpkı "Araba" bir şablon, "bizim arabamız" ise somut bir kopya gibi. Her nesne birbirinden bağımsız çalışır.

✅ Ne Öğrendik?
  • Kütüphane = hazır araç kutusu, #include ile eklenir
  • Sınıf = şablon, nesne = o şablondan üretilen gerçek kopya
  • Nesne ile sınıfın komutlarına nesne.komut() şeklinde erişilir

Kodda Doğruluk, Tasarım ve Stil

Çalışan bir kod yazmak yetmez — okunabilir, düzgün bir kod yazmak da şart. Yarışmada kodu sen yazarsın ama hata ayıklarken, üstüne bir şey eklerken ya da jüri incelerken bu fark büyük önem taşır.

1. Anlamlı İsimlendirme

❌ Kötü

int x = 15;
int y = 200;
void f1() { }

✅ İyi

int engelMesafesi = 15;
uint8_t normalHiz = 200;
void engeldeDur() { }

2. Girintileme (Indentation)

Her iç blok bir tab (ya da 2 boşluk) içeri çekilmeli. Arduino IDE bunu otomatik düzenleyebilir (Ctrl+T).

❌ Girintisiz

if (mesafe < 10) {
motorlariDurdur();
if (solSerbest) {
solaDon();
}
}

✅ Girintili

if (mesafe < 10) {
  motorlariDurdur();
  if (solSerbest) {
    solaDon();
  }
}

3. Yorum Satırları — Ama Abartma

Her satıra yorum yazma; açıklamaya ihtiyaç duyan karmaşık yerleri açıkla. Fonksiyonların üstüne ne yaptığını yaz.

Arduino C++ — Yorum Örneği 
// Motor hızlarını dengele: sol motor sağdan %10 güçlü
// çünkü mekanik asimetri var
void dengeliyIleriGit(uint8_t hiz) {
  analogWrite(SOL_MOTOR, hiz * 0.9);  // sol daha yavaş
  analogWrite(SAG_MOTOR, hiz);
}

4. Sabitleri #define veya const ile Tanımla

Kodun içinde sihirli sayılar bırakma. 200 yerine NORMAL_HIZ yazmak kodu hem anlaşılır hem değiştirilebilir yapar.

5. Her Fonksiyon Tek İş Yapsın

herSeyi yap() diye bir fonksiyon olmamalı. Fonksiyonları küçük ve odaklı tut: mesafeOku(), sagaDon(), cizgiKontrolEt() gibi.

✅ Temiz Kod Altın Kuralları
  • İsimler ne yaptığını anlatmalı
  • Girintileme tutarlı olmalı (Ctrl+T ile otomatik düzelt)
  • Yorum: karmaşık yerlerde açıkla, her satıra yazma
  • Sihirli sayı yok — sabitler için #define kullan
  • Fonksiyonlar kısa ve tek amaçlı olsun

4'lü Çizgi İzleme Modülü

Bu modülde 4 adet kızılötesi (IR) sensör bulunur. Her sensör altındaki zemine bakar: siyah çizgi gördüğünde HIGH (ya da LOW — modül tipine göre değişir), beyaz zemin gördüğünde ise tersini verir.

💡 Hangi Değer Hangi Anlama Geliyor?

Modülüne göre siyah çizgide HIGH veya LOW alabilirsin. Bağlamadan önce Serial.println() ile test et ve hangi değerin çizgiyi temsil ettiğini not et.

Arduino C++ — 4'lü Çizgi Modülü 
// Pin tanımları
#define CIZGI_SOL_DIS   2
#define CIZGI_SOL_IC    3
#define CIZGI_SAG_IC    4
#define CIZGI_SAG_DIS   5

#define CIZGI_VAR  HIGH   // modülüne göre değiştir

void setup() {
  pinMode(CIZGI_SOL_DIS, INPUT);
  pinMode(CIZGI_SOL_IC,  INPUT);
  pinMode(CIZGI_SAG_IC,  INPUT);
  pinMode(CIZGI_SAG_DIS, INPUT);
  Serial.begin(9600);
}

void loop() {
  bool solDis = digitalRead(CIZGI_SOL_DIS) == CIZGI_VAR;
  bool solIc  = digitalRead(CIZGI_SOL_IC)  == CIZGI_VAR;
  bool sagIc  = digitalRead(CIZGI_SAG_IC)  == CIZGI_VAR;
  bool sagDis = digitalRead(CIZGI_SAG_DIS) == CIZGI_VAR;

  if (solIc && sagIc) {
    // Her iki iç sensör çizgide → düz git
    ileriGit(NORMAL_HIZ);
  } else if (solIc && !sagIc) {
    // Sadece sol iç sensör çizgide → sola dön
    solaDon(DONUS_HIZ);
  } else if (!solIc && sagIc) {
    // Sadece sağ iç sensör çizgide → sağa dön
    sagaDon(DONUS_HIZ);
  } else if (solDis) {
    // Çok sola kaydı, sert sol dönüş
    solaDon(SERT_DONUS_HIZ);
  } else if (sagDis) {
    // Çok sağa kaydı, sert sağ dönüş
    sagaDon(SERT_DONUS_HIZ);
  } else {
    // Çizgi kayboldu → yavaşla
    yavasGit(YAVASH_HIZ);
  }
}
✅ Ne Öğrendik?
  • 4 sensör = 4 dijital pin, hepsi INPUT olarak ayarlanır
  • Her sensörü ayrı bool değişkene atamak kodu okunaklı yapar
  • if/else if zinciri ile hangi sensörde çizgi varsa dönüş yapılır
  • Modülüne göre siyah çizgi HIGH veya LOW olabilir — test et!

L298N ile DC Motor Sürme

L298N, Arduino'nun küçük çıkışını DC motorlar için gereken büyük güce dönüştüren bir motor sürücü kartıdır. Arduino motorları doğrudan süremez — L298N aracı gerekir.

Pim Bağlantısı

L298N PiniBağlandığı YerGörevi
IN1, IN2Arduino dijital pinSol motor yön kontrolü
IN3, IN4Arduino dijital pinSağ motor yön kontrolü
ENAArduino PWM pin (~)Sol motor hız (0–255)
ENBArduino PWM pin (~)Sağ motor hız (0–255)
🧠 Yön Nasıl Belirlenir?

IN1=HIGH, IN2=LOW → ileri  |  IN1=LOW, IN2=HIGH → geri  |  IN1=LOW, IN2=LOW → dur. ENA pinine analogWrite(hız) ile hızı belirlersin.

Arduino C++ — L298N Motor Kontrolü 
// Pin tanımları
#define IN1  6
#define IN2  7
#define IN3  8
#define IN4  9
#define ENA  5   // PWM pin!
#define ENB  10  // PWM pin!

#define NORMAL_HIZ    200
#define DONUS_HIZ     160
#define SERT_DONUS    200

void setup() {
  pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT);
  pinMode(IN3, OUTPUT); pinMode(IN4, OUTPUT);
  pinMode(ENA, OUTPUT); pinMode(ENB, OUTPUT);
}

// İleri git
void ileriGit(uint8_t hiz) {
  analogWrite(ENA, hiz); analogWrite(ENB, hiz);
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

// Geri git
void geriGit(uint8_t hiz) {
  analogWrite(ENA, hiz); analogWrite(ENB, hiz);
  digitalWrite(IN1, LOW); digitalWrite(IN2, HIGH);
  digitalWrite(IN3, LOW); digitalWrite(IN4, HIGH);
}

// Sola dön (sağ motor ileri, sol motor dur)
void solaDon(uint8_t hiz) {
  analogWrite(ENA, 0);   // sol motor dur
  analogWrite(ENB, hiz);  // sağ motor ileri
  digitalWrite(IN1, LOW);  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

// Sağa dön
void sagaDon(uint8_t hiz) {
  analogWrite(ENA, hiz);  // sol motor ileri
  analogWrite(ENB, 0);   // sağ motor dur
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW);  digitalWrite(IN4, LOW);
}

// Tamamen dur
void motorlariDurdur() {
  analogWrite(ENA, 0); analogWrite(ENB, 0);
  digitalWrite(IN1, LOW); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW); digitalWrite(IN4, LOW);
}
✅ Ne Öğrendik?
  • L298N, Arduino ile DC motorlar arasında güç köprüsüdür
  • IN1/IN2 yönü, ENA hızı belirler (PWM)
  • Motor fonksiyonlarını ayrı tanımlamak loop()'u temiz tutar

HY-SRF05 Ultrasonik Mesafe Sensörü

HY-SRF05, ultrases dalgası göndererek önündeki engele olan mesafeyi ölçer. Yarasa gibi — ses gider, geri döner, süreyi ölç, mesafeyi hesapla.

Pin Bağlantısı

Sensör PiniBağlantıGörevi
VCC5VGüç
GNDGNDToprak
TRIGArduino dijital pinSes dalgası tetikle
ECHOArduino dijital pinGeri dönen dalgayı al
OUTBağlanmayabilirEşik modunda kullanılır

HY-SRF05'i iki şekilde kullanabilirsin: kütüphanesiz (elle hesap) veya NewPing kütüphanesiyle. Kütüphane kullanmak çok daha kolaydır:

Arduino C++ — NewPing Kütüphanesi ile 
#include <NewPing.h>       // 1. Kütüphaneyi ekle

#define TRIG_PIN  11
#define ECHO_PIN  12
#define MAX_MESAFE 200    // cm cinsinden maksimum ölçüm

// 2. NewPing sınıfından "sonar" adında nesne oluştur
NewPing sonar(TRIG_PIN, ECHO_PIN, MAX_MESAFE);

void setup() {
  Serial.begin(9600);
}

void loop() {
  float mesafe = sonar.ping_cm();  // cm cinsinden mesafeyi al

  if (mesafe == 0) {
    Serial.println("Engel yok veya çok uzak");
  } else {
    Serial.print("Mesafe: ");
    Serial.print(mesafe);
    Serial.println(" cm");
  }

  delay(100);
}
💡 NewPing Kütüphanesini Eklemek

Arduino IDE → Araçlar → Kütüphane Yöneticisi → "NewPing" ara → Yükle. Kurulumdan sonra #include <NewPing.h> ile kullanmaya başlarsın.

⚠️ Dikkat

ping_cm() mesafe çok büyük veya hiç engel yoksa 0 döndürür. Kodunda bunu mutlaka kontrol et!

Kütüphanesiz Kullanım (Referans için)

Arduino C++ — Elle Hesap 
#define TRIG 11
#define ECHO 12

float mesafeOku() {
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);      // 10µs tetik sinyali
  digitalWrite(TRIG, LOW);

  long sure = pulseIn(ECHO, HIGH);  // geri dönüş süresi (µs)
  return sure * 0.034 / 2;           // cm'ye çevir
}
✅ Ne Öğrendik?
  • HY-SRF05 → ses dalgasıyla mesafe ölçer (TRIG tetikle, ECHO oku)
  • NewPing kütüphanesi ile tek satırda mesafe alınır
  • Nesne oluşturma: NewPing sonar(TRIG, ECHO, MAX);
  • Sonuç 0 ise engel yok ya da çok uzak demektir

QTR-8A Analog Çizgi Sensörü

QTR-8A, 8 adet analog IR sensörü olan gelişmiş bir çizgi sensörü kartıdır. 4'lü dijital modülden farkı: her sensör 0–1023 arasında sürekli değer verir. Bu sayede robotun çizgiden ne kadar saptığını daha hassas anlayabilirsin.

💡 Neden QTR-8A Daha Hassas?

Dijital modül "var/yok" der. QTR-8A "tam üstündeyim / biraz saptım / çok saptım" diyebilir. Bu fark, özellikle virajlarda düzgün gitmek için büyük avantaj sağlar.

QTRSensors Kütüphanesi ile Kullanım

Pololu'nun resmi kütüphanesi kurulunca kullanımı kolaylaşır: Arduino IDE → Kütüphane Yöneticisi → "QTRSensors" ara → Yükle.

Arduino C++ — QTR-8A Temel Kullanım 
#include <QTRSensors.h>

// QTRSensors sınıfından nesne oluştur
QTRSensors qtr;

#define SENSOR_SAYISI  8
uint16_t sensorDegerleri[SENSOR_SAYISI];  // 8 sensör verisi

void setup() {
  // Sensörlerin bağlı olduğu analog pinleri tanımla
  qtr.setTypeAnalog();
  qtr.setSensorPins((const uint8_t[]){A0,A1,A2,A3,A4,A5,A6,A7}, SENSOR_SAYISI);

  Serial.begin(9600);

  // Kalibrasyon: robotu çizginin üzerinde ileri geri sürükle
  Serial.println("Kalibrasyon başlıyor...");
  for (int i = 0; i < 200; i++) {
    qtr.calibrate();
    delay(20);
  }
  Serial.println("Kalibrasyon tamamlandı!");
}

void loop() {
  // Kalibre edilmiş pozisyonu oku (0 = en sol, 7000 = en sağ)
  uint16_t pozisyon = qtr.readLineBlack(sensorDegerleri);

  // Merkez 3500 → hatayı hesapla
  int hata = pozisyon - 3500;

  Serial.print("Pozisyon: "); Serial.print(pozisyon);
  Serial.print("  Hata: ");   Serial.println(hata);

  // Basit düzeltme: hata büyükse daha sert dön
  if (hata < -1000) {
    solaDon(DONUS_HIZ);
  } else if (hata > 1000) {
    sagaDon(DONUS_HIZ);
  } else {
    ileriGit(NORMAL_HIZ);
  }
}
🧠 Kalibrasyon Neden Gerekli?

Her zemin farklıdır — açık renk tahta mı, beyaz kağıt mı, mat yüzey mi? Kalibrasyon sırasında sensörler "bu zemin için en koyu değer nedir, en açık nedir?" diye öğrenir. Kalibrasyon sonrası değerler 0–1000 arasına normalize edilir. Her yarışma öncesi kalibrasyon yapılmalıdır!

✅ Ne Öğrendik?
  • QTR-8A, 8 analog sensör ile çizgi konumunu hassas ölçer
  • Kütüphane ile kalibrasyon yapılır, readLineBlack() pozisyon verir
  • Pozisyon 0–7000: 3500 merkez, altı sola sapma, üstü sağa sapma
  • PID yerine basit hata kontrolü ile de çalışabilir

Mini Proje: Engelden Kaçan Çizgi İzleyen Robot

Şimdiye kadar öğrendiklerinin hepsini birleştirelim. Bu kod, 4'lü çizgi modülü ile çizgiyi izleyen, HY-SRF05 ile önünde engel varsa duran, L298N ile iki DC motoru süren bir robot için yazılmıştır.

Arduino C++ — Tam Robot Kodu 
// ═══════════════════════════════════════════
//  MEB Robot Tasarla Çalıştır — Temel Kod
//  Çizgi İzleme + Engel Algılama
// ═══════════════════════════════════════════

#include <NewPing.h>

// ─── Motor Pinleri ───
#define IN1  6
#define IN2  7
#define IN3  8
#define IN4  9
#define ENA  5
#define ENB  10

// ─── Çizgi Sensörü Pinleri ───
#define CIZGI_SOL_DIS  2
#define CIZGI_SOL_IC   3
#define CIZGI_SAG_IC   4
#define CIZGI_SAG_DIS  A3

// ─── Mesafe Sensörü Pinleri ───
#define TRIG_PIN   11
#define ECHO_PIN   12
#define MAX_MESAFE 200

// ─── Hız Sabitleri ───
#define NORMAL_HIZ     200
#define DONUS_HIZ      160
#define SERT_DONUS     200

// ─── Engel Eşiği ───
#define ENGEL_MESAFE   15   // cm

// ─── Hangi değer çizgi anlamına geliyor? ───
#define CIZGI_VAR  HIGH    // modülüne göre değiştir

// ─── Nesne Oluşturma ───
NewPing sonar(TRIG_PIN, ECHO_PIN, MAX_MESAFE);

// ════════════════════════════════════════════
//  SETUP
// ════════════════════════════════════════════
void setup() {
  // Motor pinleri çıkış
  pinMode(IN1, OUTPUT); pinMode(IN2, OUTPUT);
  pinMode(IN3, OUTPUT); pinMode(IN4, OUTPUT);
  pinMode(ENA, OUTPUT); pinMode(ENB, OUTPUT);

  // Çizgi sensörü pinleri giriş
  pinMode(CIZGI_SOL_DIS, INPUT);
  pinMode(CIZGI_SOL_IC,  INPUT);
  pinMode(CIZGI_SAG_IC,  INPUT);
  pinMode(CIZGI_SAG_DIS, INPUT);

  Serial.begin(9600);
  Serial.println("Robot hazır!");
}

// ════════════════════════════════════════════
//  LOOP — Ana Döngü
// ════════════════════════════════════════════
void loop() {
  float mesafe = sonar.ping_cm();

  // Mesafe 0 ise engel yok (çok uzak)
  if (mesafe == 0) mesafe = 999;

  if (mesafe < ENGEL_MESAFE) {
    // ── Engel var: dur ve geri git ──
    motorlariDurdur();
    delay(500);
    geriGit(DONUS_HIZ);
    delay(600);
    sagaDon(SERT_DONUS);
    delay(400);
  } else {
    // ── Engel yok: çizgiyi izle ──
    cizgiIzle();
  }
}

// ════════════════════════════════════════════
//  FONKSİYONLAR
// ════════════════════════════════════════════

// Çizgi sensörünü oku ve motoru yönlendir
void cizgiIzle() {
  bool solDis = digitalRead(CIZGI_SOL_DIS) == CIZGI_VAR;
  bool solIc  = digitalRead(CIZGI_SOL_IC)  == CIZGI_VAR;
  bool sagIc  = digitalRead(CIZGI_SAG_IC)  == CIZGI_VAR;
  bool sagDis = digitalRead(CIZGI_SAG_DIS) == CIZGI_VAR;

  if (solIc && sagIc) {
    ileriGit(NORMAL_HIZ);
  } else if (solIc && !sagIc) {
    solaDon(DONUS_HIZ);
  } else if (!solIc && sagIc) {
    sagaDon(DONUS_HIZ);
  } else if (solDis) {
    solaDon(SERT_DONUS);
  } else if (sagDis) {
    sagaDon(SERT_DONUS);
  } else {
    ileriGit(DONUS_HIZ);  // çizgi kayboldu, yavaş ilerle
  }
}

void ileriGit(uint8_t hiz) {
  analogWrite(ENA, hiz); analogWrite(ENB, hiz);
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

void geriGit(uint8_t hiz) {
  analogWrite(ENA, hiz); analogWrite(ENB, hiz);
  digitalWrite(IN1, LOW); digitalWrite(IN2, HIGH);
  digitalWrite(IN3, LOW); digitalWrite(IN4, HIGH);
}

void solaDon(uint8_t hiz) {
  analogWrite(ENA, 0);  analogWrite(ENB, hiz);
  digitalWrite(IN1, LOW);  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

void sagaDon(uint8_t hiz) {
  analogWrite(ENA, hiz); analogWrite(ENB, 0);
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW);  digitalWrite(IN4, LOW);
}

void motorlariDurdur() {
  analogWrite(ENA, 0); analogWrite(ENB, 0);
  digitalWrite(IN1, LOW); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW); digitalWrite(IN4, LOW);
}
💡 Bu Kodda Neler Kullanıldı?
  • #define ile tüm sabitler başta tanımlandı
  • NewPing kütüphanesinden sonar nesnesi oluşturuldu
  • setup()'ta pinler hazırlandı
  • loop()'ta önce engel kontrolü, sonra çizgi izleme yapıldı
  • Her davranış ayrı fonksiyona ayrıldı — loop() temiz kaldı
  • if / else if / else zinciriyle karar verildi
  • uint8_t hız parametreleri için kullanıldı
