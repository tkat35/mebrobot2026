Robotu tanıyalım

Giriş

Çizgi takip robotu, yere çizilmiş siyah bir çizgiyi takip etmek için programlanmış küçük bir araçtır. İki temel parçası vardır: sensörler (çizgiyi gören gözler) ve motorlar (hareketi sağlayan kaslar).

Robotumuzda 4 adet infrared sensör var. Bu sensörler yere kızılötesi ışık gönderir. Yüzey beyazsa ışık geri yansır → sensör 0 der. Yüzey siyahsa ışık yutulur, geri gelmez → sensör 1 der.

Ayrıca iki motorumuz var: sol tekerlek ve sağ tekerlek. İkisi aynı hızda dönerse robot düz gider. Biri yavaşlarsa veya tersine dönerse robot o tarafa döner.

💡 Kolay anlamak için: Dört parmağını yere uzat ve gözlerini kapat. Hangisi siyah çizgiyi hissederse o taraf "1" diyor, hissetmeyenler "0" diyor. Robot da sensörleriyle tam böyle hissediyor!

1

Temel Kod — Pin ve Hız Tanımlamaları

Yapı Taşı

Bu kod doğrudan "zeka" değildir. Bu, robotun anayasasıdır. Hangi kablo hangi işi yapıyor, motorlar hangi hızda dönecek — bu temel kurallar burada yazılır. Tüm diğer algoritmalar bu yapının üzerine inşa edilir.

const int dediğimizde "bu sayı hiç değişmeyecek, sabittir" diyoruz. const = sabit, int = tam sayı anlamına gelir.

setup() fonksiyonu robot açılınca sadece bir kez çalışır. Pinlerin giriş mi çıkış mı olduğunu burada söylüyoruz.

loop() ise robot çalıştığı sürece sürekli tekrar eder — saniyede yüzlerce kez! Robot bu fonksiyon içinde sensörleri okur ve motorlara emir verir.

1

Motor pinleri — Arduino'dan motora giden yol

ENA ve ENB hız kontrolü yapar (0–255 arası değer alır). IN1/IN2 sol motorun yönünü, IN3/IN4 sağ motorun yönünü belirler. HIGH+LOW = ileri, LOW+HIGH = geri.

2

Sensör pinleri — motordan Arduino'ya gelen bilgi

Her sensör bir pin'e bağlı. digitalRead(pin) ile o sensörün 0 mı 1 mi söylediğini öğreniyoruz. INPUT yani GİRİŞ: bilgi dışarıdan içeriye geliyor.

3

PWM ile hız kontrolü

analogWrite(ENA, 160) → sol motoru %63 hızda döndür. 0 = dur, 128 = yarı hız, 255 = tam hız. "Analog" gibi görünse de aslında çok hızlı açıp kapayarak hız simülasyonu yapar — buna PWM (Pulse Width Modulation) denir.

// ═══════════════════════════════════════════════
// MOTOR PİN TANIMLAMALARI
// L298N motor sürücüsüne bağlı pinler
// ═══════════════════════════════════════════════
const int ENA = 6;   // Sol motorun hız pini  (PWM — tilde ~ olan pin)
const int IN1 = 7;   // Sol motor yön pini 1
const int IN2 = 8;   // Sol motor yön pini 2
const int IN3 = 9;   // Sağ motor yön pini 1
const int IN4 = 10;  // Sağ motor yön pini 2
const int ENB = 11;  // Sağ motorun hız pini  (PWM)

// SENSÖR PİN TANIMLAMALARI
const int S_SOL_DIS = 2; // Sol Dış sensör
const int S_SOL_IC  = 3; // Sol İç sensör
const int S_SAG_IC  = 4; // Sağ İç sensör
const int S_SAG_DIS = 5; // Sağ Dış sensör

// HIZ AYARLARI (0 = dur, 255 = tam gaz)
// Bu değerleri robotuna göre değiştir!
int temelHiz  = 160; // Düz giderken hız (~%63)
int donusHizi = 120; // Dönüşlerde hız (~%47)

// ═══════════════════════════════════════════════
// SETUP — Sadece bir kez çalışır
// ═══════════════════════════════════════════════
void setup() {
  // Motorlar ÇIKIŞ — Arduino onlara emir veriyor
  pinMode(ENA, OUTPUT);
  pinMode(IN1, OUTPUT);
  pinMode(IN2, OUTPUT);
  pinMode(IN3, OUTPUT);
  pinMode(IN4, OUTPUT);
  pinMode(ENB, OUTPUT);

  // Sensörler GİRİŞ — bilgi dışarıdan içeriye geliyor
  pinMode(S_SOL_DIS, INPUT);
  pinMode(S_SOL_IC,  INPUT);
  pinMode(S_SAG_IC,  INPUT);
  pinMode(S_SAG_DIS, INPUT);
}

// ═══════════════════════════════════════════════
// TEMEL MOTOR FONKSİYONLARI
// Bu fonksiyonlar tüm algoritmalarda kullanılır
// ═══════════════════════════════════════════════
void ilerle() {
  analogWrite (ENA, temelHiz); // Sol motor hızı
  digitalWrite(IN1, HIGH);      // Sol motor ileri
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH);      // Sağ motor ileri
  digitalWrite(IN4, LOW);
  analogWrite (ENB, temelHiz); // Sağ motor hızı
}

void hafifSol() {
  analogWrite (ENA, donusHizi / 2); // Sol motor YAVAŞ
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH);
  digitalWrite(IN4, LOW);
  analogWrite (ENB, temelHiz);       // Sağ motor NORMAL
}

void sertSol() {
  analogWrite (ENA, donusHizi);
  digitalWrite(IN1, LOW);            // Sol motor GERİ!
  digitalWrite(IN2, HIGH);
  digitalWrite(IN3, HIGH);           // Sağ motor ileri
  digitalWrite(IN4, LOW);
  analogWrite (ENB, donusHizi);
}

void hafifSag() {
  analogWrite (ENA, temelHiz);       // Sol motor NORMAL
  digitalWrite(IN1, HIGH);
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH);
  digitalWrite(IN4, LOW);
  analogWrite (ENB, donusHizi / 2); // Sağ motor YAVAŞ
}

void sertSag() {
  analogWrite (ENA, donusHizi);
  digitalWrite(IN1, HIGH);           // Sol motor ileri
  digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW);
  digitalWrite(IN4, HIGH);           // Sağ motor GERİ!
  analogWrite (ENB, donusHizi);
}

void dur() {
  digitalWrite(IN1, LOW); digitalWrite(IN2, LOW);
  digitalWrite(IN3, LOW); digitalWrite(IN4, LOW);
  analogWrite (ENA, 0);
  analogWrite (ENB, 0);
}

Mantıksal Kontrol — "Eğer... ise yap" Robotu

Temel Algoritma

Bu en basit yöntemdir. Robot sensörlere bakıp "eğer şuysa → şunu yap" der. Sanki bir trafik polisi gibi — sadece anlık duruma göre karar verir, geçmişe ya da geleceğe bakmaz.

4 sensörümüz var, her biri 0 veya 1 olabilir. Bu 2×2×2×2 = 16 farklı durum demektir. Biz en önemli olanları seçip tanımladık, geri kalanlar için "yavaşça ilerle" dedik.

Avantajı: Anlaması ve yazması çok kolay. Dezavantajı: Kaba tepkiler verir — hafif kaydıkta da sert döndüğü oluşuyor.

🎮 Oyun gibi düşün: Bu algoritma bir video oyunundaki en basit "yapay zeka" gibi çalışır. "Eğer düşman solda → sağa kaç, eğer düşman sağda → sola kaç." Basit ama hızlı çalışır. Karmaşık durumları iyi çözemez.

// ═══════════════════════════════════════════════
// MANTIKSAL KONTROL — Tam loop() fonksiyonu
// setup() ve motor fonksiyonları Bölüm 1'deki ile aynı
// ═══════════════════════════════════════════════
void loop() {
  // Her döngüde sensörleri oku
  int s1 = digitalRead(S_SOL_DIS);
  int s2 = digitalRead(S_SOL_IC);
  int s3 = digitalRead(S_SAG_IC);
  int s4 = digitalRead(S_SAG_DIS);

  // MANTIK TABLOSU — hangi durum → ne yap
  // && = "ve" demek, her iki koşul da doğru olmalı

  if (s1==0 && s2==1 && s3==1 && s4==0) {
    ilerle();         // Mükemmel — düz git
  }
  else if (s1==0 && s2==1 && s3==0 && s4==0) {
    hafifSol();       // Biraz sağa kaydık, hafifçe sola dön
  }
  else if (s1==1 && s2==0 && s3==0 && s4==0) {
    sertSol();        // Çok sağa kaydık, hızlıca sola dön!
  }
  else if (s1==0 && s2==0 && s3==1 && s4==0) {
    hafifSag();       // Biraz sola kaydık, hafifçe sağa dön
  }
  else if (s1==0 && s2==0 && s3==0 && s4==1) {
    sertSag();        // Çok sola kaydık, hızlıca sağa dön!
  }
  else if (s1==0 && s2==0 && s3==0 && s4==0) {
    dur();            // Hiçbir şey yok — dur
  }
  else if (s1==1 && s2==1 && s3==1 && s4==1) {
    dur();            // Hepsi görüyor — bitiş çizgisi, dur!
  }
  else {
    ilerle();         // Bilinmeyen durum — yavaşça devam et
  }
}
Bu algoritmada ne eksik? Robot "hafif kayma" ile "çok kayma" arasındaki farkı bilmiyor. Hafif sol ve sert sol arasında sadece iki seçenek var. Gerçekte ise sonsuz sayıda ara değer olabilir — işte P algoritması bu sorunu çözüyor!
3

P Kontrolcüsü — "Ne kadar kaymışım?"

P Algoritması

Mantıksal kontrolde robot sadece "sola mı sağa mı?" diyordu. Ama ne kadar sola gittiğini bilmiyordu!

P (Proportional = Orantılı) kontrolcüde robot şunu sorar: "Çizgiden ne kadar uzaklaştım?" Ve uzaklaştığı kadar düzeltme yapar. Az kaydıysa az, çok kaydıysa çok döner.

Bu yöntemde sayısal bir "hata" değeri hesaplıyoruz. Hata sıfır = robot tam ortada. Hata büyük = robot çok kaymış. Bu hatayı Kp ile çarpıp motor hızına ekliyoruz.

🚗 Araba direksiyonu gibi: Yolun ortasındayken direksiyonu az çevirirsin. Yolun kenarına gidince çok çevirirsin. P kontrolcü de tam böyle — hata ne kadar büyükse, düzeltme o kadar büyük olur!
P

Proportional (Orantılı)

Hatanın büyüklüğüne göre düzeltme yapar. Hata 2 ise 2×Kp kadar, hata 5 ise 5×Kp kadar düzeltir. Kp katsayısını sen belirlersin.

E

Hata (Error) nedir?

Sensörlerden hesaplanan "çizgiden ne kadar uzaklaştım" değeri. Sıfır = mükemmel, artı = sağa kaydım, eksi = sola kaydım.

Hata hesaplama
hata = (s1 × 2) + (s2 × 1) − (s3 × 1) − (s4 × 2)
Sol sensörler pozitif (+), sağ sensörler negatif (−) → sıfır = tam ortada

Hata formülü nasıl çalışır? İç sensörlere 1, dış sensörlere 2 ağırlık veriyoruz. Çünkü dış sensörlerin görmesi, daha şiddetli bir sapma demektir.

Örnek: s2=1, s3=0 → hata = 1. Robot biraz sağa kaymış.
Örnek: s1=1 → hata = 2. Robot çok sağa kaymış.
Örnek: s4=1 → hata = −2. Robot çok sola kaymış.

// ═══════════════════════════════════════════════
// P KONTROLCÜSÜ — Tam Kod
// Pin tanımlamaları ve motor fonksiyonları aynı
// ═══════════════════════════════════════════════
const int ENA=6, IN1=7, IN2=8;
const int IN3=9, IN4=10, ENB=11;
const int S1=2, S2=3, S3=4, S4=5;

float Kp       = 20.0; // P katsayısı — deneyerek bulursun!
int   temelHiz = 160;  // Ortalama ilerleme hızı

void setup() {
  pinMode(ENA,OUTPUT); pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT); pinMode(ENB,OUTPUT);
  pinMode(S1,INPUT); pinMode(S2,INPUT);
  pinMode(S3,INPUT); pinMode(S4,INPUT);
}

void loop() {
  // ADIM 1: Sensörleri oku
  int s1=digitalRead(S1), s2=digitalRead(S2);
  int s3=digitalRead(S3), s4=digitalRead(S4);

  // ADIM 2: Hatayı hesapla
  // Sol sensörler + (sağa kaydık demek → sola dön)
  // Sağ sensörler - (sola kaydık demek → sağa dön)
  // İç sensörler 1 ağırlık, dış sensörler 2 ağırlık
  int hata = (s1*2) + (s2*1) - (s3*1) - (s4*2);

  // ADIM 3: P düzeltmesini hesapla
  // hata büyükse düzeltme büyük, hata küçükse düzeltme küçük
  int duzeltme = Kp * hata;

  // ADIM 4: Motor hızlarını ayarla
  // Hata pozitif (sağa kaydık):
  //   → sol motor hızlanır, sağ motor yavaşlar → robot sola döner
  // Hata negatif (sola kaydık):
  //   → sağ motor hızlanır, sol motor yavaşlar → robot sağa döner
  int solHiz = constrain(temelHiz - duzeltme, 0, 255);
  int sagHiz = constrain(temelHiz + duzeltme, 0, 255);

  // ADIM 5: Motorlara hız ver
  analogWrite (ENA, solHiz);
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
  analogWrite (ENB, sagHiz);
}

// constrain(deger, min, max):
// değeri sınırlar içinde tutar.
// Örnek: constrain(300, 0, 255) → 255 döndürür
// Örnek: constrain(-50, 0, 255) → 0 döndürür
Kp'yi nasıl ayarlarım? Kp çok küçük olursa robot yavaş tepki verir ve çizgiden çıkabilir. Kp çok büyük olursa robot sağa-sola sallanır (titrer). Doğru değeri bulmak için robotunu gerçek pistte çalıştırıp deneyerek ayarlamalısın — bu işleme tuning (ayarlama) denir!

Kp çok küçük Robot çizgiden çıkıyor, tepkiler çok yavaş Kp değerini artır (örn: 10 → 20)
Kp çok büyük Robot sağa-sola titriyor, yılan gibi gidiyor Kp değerini azalt (örn: 50 → 25)
Kp doğru Robot akıcı ve kararlı şekilde çizgiyi takip ediyor Tebrikler! Şimdi PD'yi dene

PD Kontrolcüsü — "Ne kadar hızlı kayıyorum?"

PD Algoritması

P kontrolcü çok iyiydi ama bir sorunu vardı: sadece şu anki hataya bakıyor, hatanın değişimine bakmıyordu.

D (Derivative = Türev) terimi "hata ne hızda değişiyor?" diye sorar. Hata hızla büyüyorsa, çok daha sert bir düzeltme gerekir. Hata zaten azalıyorsa, fazla müdahaleye gerek yok!

D terimi, bir önceki döngüdeki hatayı hatırlayarak şimdiki hatayla karşılaştırır. Aradaki fark ne kadar büyükse, D'nin katkısı o kadar büyük olur.

⚽ Kaleci gibi düşün: Kaleci sadece topun nerede olduğuna değil, hangi yöne gittiğine de bakıyor. Top hızla köşeye gidiyorsa erken atlar. P sadece topu görür, D topun hızını ve yönünü de görür. İkisi birlikte çok daha iyi bir kaleci yapar!
P

Mevcut Hata

Şu an çizgiden ne kadar uzaktayım? Büyük hata → büyük P düzeltmesi. Kp ile çarpılır.

D

Hatanın Değişimi

Hata önceki ölçüme göre ne kadar değişti? Hızla kötüleşiyorsa daha sert tepki ver! Kd ile çarpılır.

PD formülü
düzeltme = (Kp × hata) + (Kd × (hata − öncekiHata))
(hata − öncekiHata) = türev terimi = hatanın değişme hızı
1

Sensörleri oku ve hatayı hesapla (P ile aynı)

Sensörlerden değerleri al. Ağırlıklı hata formülüyle sayısal bir "hata" değeri üret.

2

D terimini hesapla — önceki hatayla karşılaştır

hata - oncekiHata ifadesi, hatanın bu döngüde ne kadar değiştiğini verir. Bu farkı Kd ile çarp.

3

Şimdiki hatayı kaydet — bir sonraki döngü için!

oncekiHata = hata; satırı çok önemli! Olmasa bir sonraki döngüde önceki hatayı hatırlayamayız.

4

Motor hızlarını ayarla ve motorlara ver

// ═══════════════════════════════════════════════
// PD KONTROLCÜSÜ — Tam Kod
// P'den sadece 3 küçük değişiklik var!
// ═══════════════════════════════════════════════
const int ENA=6, IN1=7, IN2=8;
const int IN3=9, IN4=10, ENB=11;
const int S1=2, S2=3, S3=4, S4=5;

float Kp       = 20.0;  // P katsayısı
float Kd       = 10.0;  // D katsayısı — DEĞİŞİKLİK 1: Kd eklendi
int   temelHiz = 160;

int oncekiHata = 0;     // DEĞİŞİKLİK 2: Önceki hata değişkeni eklendi

void setup() {
  pinMode(ENA,OUTPUT); pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT); pinMode(ENB,OUTPUT);
  pinMode(S1,INPUT); pinMode(S2,INPUT);
  pinMode(S3,INPUT); pinMode(S4,INPUT);
}

void loop() {
  // ADIM 1: Sensörleri oku (aynı)
  int s1=digitalRead(S1), s2=digitalRead(S2);
  int s3=digitalRead(S3), s4=digitalRead(S4);

  // ADIM 2: Şimdiki hatayı hesapla (aynı)
  int hata = (s1*2) + (s2*1) - (s3*1) - (s4*2);

  // ADIM 3: PD düzeltmesini hesapla
  // DEĞİŞİKLİK 3: Formüle Kd*(hata-oncekiHata) eklendi
  // (hata - oncekiHata) = hatanın bu döngüde ne kadar değiştiği
  int duzeltme = (Kp * hata) + (Kd * (hata - oncekiHata));

  // ADIM 4: Bu hatayı bir sonraki döngü için sakla!
  // Bu satır olmadan D terimi çalışmaz!
  oncekiHata = hata;

  // ADIM 5: Motor hızlarını ayarla (aynı)
  int solHiz = constrain(temelHiz - duzeltme, 0, 255);
  int sagHiz = constrain(temelHiz + duzeltme, 0, 255);

  // ADIM 6: Motorlara hız ver (aynı)
  analogWrite (ENA, solHiz);
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
  analogWrite (ENB, sagHiz);
}
P'den PD'ye geçişte ne değişti? Sadece 3 şey: (1) float Kd = 10.0; eklendi, (2) int oncekiHata = 0; eklendi, (3) formülde + Kd * (hata - oncekiHata) eklendi. Gerisi tamamen aynı! PD, P'nin üzerine kurulmuş geliştirilmiş bir versiyonudur.

QTR-8A ile Çizgi Takip

QTR-8A

QTR-8A nedir? Pololu firmasının ürettiği, üzerinde 8 adet kızılötesi sensör bulunan bir şerittir. Her sensör hem kızılötesi ışık gönderir hem de yansıyan ışığı ölçer ve bir analog değer (0–1023) üretir. Sayı büyükse koyu zemin, küçükse açık zemin demektir.

4'lü dijital sensörden farkı: her sensör sadece "var / yok" değil, ne kadar var bilgisini verir. Pololu'nun kütüphanesi bu değerleri okuyup robot çizginin tam olarak nerede olduğunu gösteren tek bir konum değerine (0–5000) dönüştürür — hem kalibrasyon hem hesap otomatik!

💡 Analojiyle düşün: Dijital sensör parmağınla siyahı hissedip "evet/hayır" demek gibi. QTR-8A ise göz kapağını kısmen açıp "az mı var, çok mu var?" diye bakmak gibi — çok daha fazla bilgi! Kütüphane de bu bilgiyi "robot çizginin kaç mm sağındasın?" sorusuna çeviriyor.

Neden 6 sensör? QTR-8A'nın 8 sensörü var ama Arduino UNO'da sadece 6 analog pin (A0–A5) bulunuyor. Bu nedenle en soldaki (S0) ve en sağdaki (S7) sensörleri devre dışı bırakıp ortadaki 6 sensörü (S1–S6) kullanıyoruz. Kütüphaneye de kaç sensör kullandığımızı söylüyoruz.

Kütüphane kurulumu: Arduino IDE'de Sketch → Include Library → Manage Libraries menüsüne gir, arama kutusuna "QTRSensors" yaz ve Pololu'nun kütüphanesini yükle. Bu kütüphane üç kritik işlemi otomatik yapar:

① Kalibrasyon: Robotu çizginin üzerinde ileri-geri hareket ettirirken kütüphane her sensörün minimum ve maksimum değerlerini öğrenir.
② Normalize okuma: Ham analog değer yerine 0–1000 arası normalize değer üretir — zemin renginden bağımsız.
③ Konum hesabı: 6 sensörü birleştirip robotun çizgiye göre konumunu 0–5000 arasında tek bir sayıyla verir. 2500 = tam orta, 0 = en solda, 5000 = en sağda.

Konum değerini (position) anlamak: 6 sensör için kütüphane 0–5000 arası değer üretir. Orta = 2500. Bu değerden 2500 çıkarırsak hata −2500 ile +2500 arasında olur: negatif = sola kaymış, pozitif = sağa kaymış. Algoritmalar bu hatayı kullanır — tıpkı 4'lü sensördeki gibi!
⚙️ Kalibrasyon neden şart? Her ortamda ışık farklıdır. Kütüphane kalibrasyon sırasında "bu sensörün gördüğü en koyu değer neydi, en açık değer neydi?" diye kaydeder. Sonra her okumayı bu aralığa göre normalize eder. Böylece güneşli pistte de, kapalı spor salonunda da aynı kod çalışır.
6.1

Mantıksal Kontrol — QTR-8A

Mantıksal

Kütüphanenin readLineBlack() fonksiyonu bize 0–5000 arası bir konum değeri verir. Biz bu değeri 2500'den çıkararak −2500 ile +2500 arası bir hata elde ederiz. Sonra bu hatayı basamaklara bölerek if/else kurallar yazarız — tıpkı 4'lü sensördeki mantık gibi ama çok daha hassas!

// ──────────────────────────────────────────────────────────
// QTR-8A ile Mantıksal Kontrol
// Kütüphane: QTRSensors (Pololu) — Library Manager'dan yükle
// Arduino UNO + L298N Motor Sürücü
// ──────────────────────────────────────────────────────────

#include <QTRSensors.h>

// ── Kütüphane nesnesi ──
QTRSensors qtr;

// ── Motor pinleri (L298N) ──
const int ENA = 10, IN1 = 7, IN2 = 8;   // Sol motor
const int ENB = 11, IN3 = 9, IN4 = 12;  // Sağ motor

const int HIZ_ILERI = 170;
const int HIZ_YAVS  = 110;
const int HIZ_SERT  = 200;

void motorSur(int sol, int sag) {
  analogWrite(ENA, constrain(sol, 0, 255));
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  analogWrite(ENB, constrain(sag, 0, 255));
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

void setup() {
  pinMode(ENA,OUTPUT); pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(ENB,OUTPUT); pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT);

  // ── Kütüphane ayarı ──
  // 6 sensör, analog tip, S1–S6 → A0–A5 pinleri
  qtr.setTypeAnalog();
  qtr.setSensorPins((uint8_t[]){A0, A1, A2, A3, A4, A5}, 6);

  // ── Kalibrasyon ──
  // Robotu 3–4 saniye çizginin üzerinde ileri-geri salla
  // (veya robotu el ile döndür, tüm sensörler çizgiyi görsün)
  // Bu sürede kütüphane her sensörün min/max değerini öğrenir
  for (int i = 0; i < 200; i++) {
    qtr.calibrate();
    delay(20);  // 200 × 20ms = 4 saniye kalibrasyon
  }
  // Kalibrasyon bitti — robot artık hazır!
}

void loop() {
  // ADIM 1: Konum oku (0 = en sol, 5000 = en sağ, 2500 = tam orta)
  // readLineBlack → siyah çizgi için kullan
  uint16_t konum = qtr.readLineBlack(nullptr);

  // ADIM 2: Hatayı hesapla (2500'den sapma)
  // negatif hata = robot sola kaymış, pozitif = sağa kaymış
  int hata = (int)konum - 2500;

  // ADIM 3: Hatanın büyüklüğüne göre mantıksal karar ver
  // Aralıklar 0–2500 ölçeğine göre belirlenmiş:
  //   ±200  → neredeyse ortada
  //   ±800  → orta kayma
  //   ±800+ → ciddi kayma

  if (hata > -200 && hata < 200) {
    // Ortada → düz git
    motorSur(HIZ_ILERI, HIZ_ILERI);

  } else if (hata < 0 && hata >= -800) {
    // Hafif sola kaymış → yavaşça sola dön
    motorSur(HIZ_YAVS, HIZ_ILERI);

  } else if (hata > 0 && hata <= 800) {
    // Hafif sağa kaymış → yavaşça sağa dön
    motorSur(HIZ_ILERI, HIZ_YAVS);

  } else if (hata < -800) {
    // Ciddi sola kayma → sert sola dön
    motorSur(0, HIZ_SERT);

  } else {
    // Ciddi sağa kayma → sert sağa dön
    motorSur(HIZ_SERT, 0);
  }
}
Kalibrasyon nasıl yapılır? setup() içindeki döngü 4 saniye çalışır. Bu 4 saniye boyunca robotu elinle çizginin üzerinde ileri-geri veya sağa-sola döndür; tüm sensörler hem siyahı hem beyazı görsün. Süre bitince robot otomatik çalışmaya başlar.
6.2

P Kontrolcü — QTR-8A

P Kontrol

Kütüphanenin verdiği 0–5000 konum değerinden ürettiğimiz hata artık −2500 ile +2500 arasında. P kontrolcü bu hatayı Kp ile çarparak motor farkını bulur. Hata büyükse büyük düzeltme, hata küçükse küçük düzeltme — otomatik ve sürekli!

4'lü sensörde hata −2 ile +2 arasındaydı; burada −2500 ile +2500 arasında. Bu yüzden Kp değeri çok küçük seçilmeli — 0.05–0.15 gibi ondalık bir sayı kullanırız.

P Kontrolcü — QTR-8A ile
konum = qtr.readLineBlack() ← 0–5000 arası
hata = konum − 2500 ← −2500 ile +2500 arası
düzeltme = Kp × hata
solMotor = temelHız − düzeltme
sağMotor = temelHız + düzeltme
Kp başlangıç değeri: 0.08 — robotu piste koy ve yavaşça artır
// ──────────────────────────────────────────────────────────
// QTR-8A ile P Kontrolcü
// Kütüphane: QTRSensors (Pololu)
// ──────────────────────────────────────────────────────────

#include <QTRSensors.h>

QTRSensors qtr;

const int ENA = 10, IN1 = 7, IN2 = 8;
const int ENB = 11, IN3 = 9, IN4 = 12;

// P kontrolcü parametreleri
// Hata aralığı −2500…+2500 olduğundan Kp çok küçük seçilir!
float Kp        = 0.08;  // Başlangıç — piste göre ayarla
int   temelHiz  = 160;
int   sensorPinleri[6];


void motorSur(int sol, int sag) {
  analogWrite(ENA, constrain(sol, 0, 255));
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  analogWrite(ENB, constrain(sag, 0, 255));
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

void setup() {
  pinMode(ENA,OUTPUT); pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(ENB,OUTPUT); pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT);

  qtr.setTypeAnalog();
  qtr.setSensorPins((uint8_t[]){A0, A1, A2, A3, A4, A5}, 6);

  // Kalibrasyon — 4 saniye robotu çizgi üzerinde gezdir
  for (int i = 0; i < 200; i++) {
    qtr.calibrate();
    delay(20);
  }
}

void loop() {
  // ADIM 1: Konum oku — kütüphane her şeyi halleder
  uint16_t konum = qtr.readLineBlack(sensorPinleri);

  // ADIM 2: Hatayı hesapla
  int hata = konum - 2500;

  // ADIM 3: P düzeltmesi
  int duzeltme = (Kp * hata);

  // ADIM 4: Motorları ayarla
  int solHiz = constrain(temelHiz - duzeltme, 0, 255);
  int sagHiz = constrain(temelHiz + duzeltme, 0, 255);

  motorSur(solHiz, sagHiz);
}

6.3

PD Kontrolcü — QTR-8A

PD Kontrol

P kontrolcüye sadece 3 satır ekliyoruz: Kd değişkeni, öncekiHata değişkeni ve formülde türev terimi. Hata hızla büyüyorsa D terimi "dur, çok sert dönüyorsun" diye frenleme yapar. Hata azalıyorsa "iyi gidiyorsun, çok düzeltme yapma" der. Sonuç: titremesiz, akıcı gidiş.

PD Kontrolcü Formülü
hata = konum − 2500
düzeltme = (Kp × hata) + (Kd × (hata − öncekiHata))
öncekiHata = hata ← bir sonraki döngü için sakla
Kp başlangıç: 0.08 — Kd başlangıç: 0.5 — ikisini birlikte ayarla
// ──────────────────────────────────────────────────────────
// QTR-8A ile PD Kontrolcü
// P koduna göre sadece 3 satır eklendi — farklı yerler işaretli
// ──────────────────────────────────────────────────────────

#include <QTRSensors.h>

QTRSensors qtr;

const int ENA = 10, IN1 = 7, IN2 = 8;
const int ENB = 11, IN3 = 9, IN4 = 12;

float Kp          = 0.08;  // Orantı katsayısı (P'deki ile aynı)
float Kd          = 0.5;   // DEĞİŞİKLİK 1: Türev katsayısı eklendi
int   temelHiz    = 160;
int   oncekiHata  = 0;    // DEĞİŞİKLİK 2: Önceki hata hafızası eklendi
int   sensorPinleri[];    


void motorSur(int sol, int sag) {
  analogWrite(ENA, constrain(sol, 0, 255));
  digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW);
  analogWrite(ENB, constrain(sag, 0, 255));
  digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW);
}

void setup() {
  pinMode(ENA,OUTPUT); pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(ENB,OUTPUT); pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT);

  qtr.setTypeAnalog();
  qtr.setSensorPins((uint8_t[]){A0, A1, A2, A3, A4, A5}, 6);

  // Kalibrasyon — 4 saniye robotu çizgi üzerinde gezdir
  for (int i = 0; i < 200; i++) {
    qtr.calibrate();
    delay(20);
  }
}

void loop() {
  // ADIM 1: Konum oku (aynı)
  uint16_t konum = qtr.readLineBlack(sensorPinleri);

  // ADIM 2: Hatayı hesapla (aynı)
  int hata = konum - 2500;

  // ADIM 3: PD düzeltmesi
  // DEĞİŞİKLİK 3: Formüle Kd × (hata − öncekiHata) eklendi
  int duzeltme = ((Kp * hata) + (Kd * (hata - oncekiHata)));

  // ADIM 4: Önceki hatayı güncelle — bu satır olmadan D terimi çalışmaz!
  oncekiHata = hata;

  // ADIM 5: Motorları ayarla (aynı)
  int solHiz = constrain(temelHiz - duzeltme, 0, 255);
  int sagHiz = constrain(temelHiz + duzeltme, 0, 255);

  motorSur(solHiz, sagHiz);
}
